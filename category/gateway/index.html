🔶上游（Upstream）

上游：应用层服务 / 节点 的抽象；

路由 / 服务 与 上游 对应关系可以为：多对一。

🔶服务（Service）

服务：某类 API 的抽象？（一组 路由 的抽象？）

对应关系：服务与上游 通常是 一对一，路由与服务 通常是 多对一。

6 Treminology

API Gateway

API 网关核心职责：

反向代理、接受所有 API 的调用、整合处理后端服务、返回相应的结果。

API 网关功能：

  • 请求转发；
  • 接口安全：身份验证、授权、速率限制；
  • 分析：监控、日志记录、跟踪；
  • 防御 SQL 注入、DDOS 攻击、XSS 等恶意攻击媒介；

10 入门篇：JIT 编译器的死穴：为什么要避免使用 NYI？

专业术语
